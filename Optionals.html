Optionals:

-handles the absence of a value. Optionals say either “there is a value, and it equals x” or “there isn’t a value at all”.

Declaration:

var perhapsInt: Int?

Unwrapping an Optional Value:

If you define a variable as optional, then to get the value from this variable, you will have to unwrap it.

Force Unwrapping:

If you are sure that an optional has a value, you can unwrap it by adding an exclamation to the end of the variable name. For instance–

if username != nil{  
Print(username!)  
}

Optional Binding:

Use an if-let statement to run some code after assuring that the optional has a value. You can also do this to define how to handle an error, for example–

var myString:String?  
myString = "Hello, Swift 4!"  
  
if  let yourString = myString {  
print("Your string has - \(yourString)")  
} else {  
print("Your string does not have a value")  
}

A guard-let statement does the same thing, except it creates a new value for the non-optional value if there is one, unlike this example where yourString cannot be used anywhere else
